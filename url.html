Term 05 — Addressing

URL

Uniform Resource Locator — the standardised address used to locate resources such as web pages, images, and files on the Internet.

📷 Add your image: images/url.jpg

DEFINITION

A Uniform Resource Locator (URL) is the complete web address used to identify and locate a specific resource on the Internet. It tells the browser what protocol to use, which server to connect to, and which specific resource to request from that server.

How It Works

A URL is made up of several parts. The protocol (https://) tells the browser how to connect. The domain name (www.example.com) identifies the server. The path (/page/about) points to a specific resource. Optional query strings (?id=1) pass extra information to the server.

When you enter a URL, your browser first asks DNS to resolve the domain name into an IP address. It then connects to that IP on the correct port and requests the resource at the given path. The server returns the resource if it exists.

Structure

protocol://domain/path?query — each part plays a specific role in locating a resource.

Protocol Part

Specifies how to connect — common values are http, https, and ftp.

Domain Part

The human-readable server name that is resolved to an IP address via DNS.

Example

https://www.example.com/page?id=1 is a complete URL with all parts included.
