Term 10 — System Design

Multitier Architecture

A client-server software design pattern that separates an application into distinct logical layers, each with specific responsibilities.

📷 Add your image: images/multitier.jpg

DEFINITION

Multitier architecture, also known as n-tier architecture, is a software design approach that divides an application into separate logical layers. Each layer handles a specific part of the application and communicates only with the layers directly above or below it.

How It Works

The most common model is three-tier architecture. The Presentation Tier is the user interface — the browser or app the user sees. The Logic Tier is the application server that processes business rules and handles requests. The Data Tier is the database server that stores and retrieves data.

When a user submits a form, the browser sends the data to the application server which applies business logic, queries the database, and sends the result back to the browser for display. Each tier can be scaled or updated independently without affecting the others.

Common Model

Three-tier: Presentation, Logic, and Data tiers working together.

Key Benefit

Each tier can be scaled, maintained, and updated independently of the others.

Example Flow

Browser sends request → App server processes it → Database returns data.

Also Called

N-tier architecture — where N represents the number of layers used.
