Term 07 — Protocol

HTTP

Hypertext Transfer Protocol — the application-layer protocol that governs how data is requested and transmitted between web browsers and web servers.

📷 Add your image: images/http.jpg

DEFINITION

Hypertext Transfer Protocol (HTTP) is the foundation of data communication on the World Wide Web. It is a stateless request-response protocol that defines how messages are formatted and transmitted between web browsers (clients) and web servers.

How It Works

A client sends an HTTP request with a method such as GET (retrieve data), POST (send data), PUT (update data), or DELETE (remove data), along with headers and an optional body. The server processes the request and returns a response with a status code and content.

HTTPS is the secure version of HTTP, using TLS/SSL encryption to protect data in transit. Modern HTTP/2 and HTTP/3 improve performance with multiplexing and faster connection handling compared to the older HTTP/1.1 standard.

Request Methods

GET, POST, PUT, and DELETE are the most commonly used HTTP methods.

Status Codes

200 OK, 301 Redirect, 404 Not Found, and 500 Server Error are common responses.

Secure Version

HTTPS encrypts communication using TLS/SSL to protect data in transit.

Current Version

HTTP/3 is the latest version, using QUIC protocol for faster performance.
